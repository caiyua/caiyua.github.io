    # 快速上手

    按照 Vuex 的观念，是把要共享的数据放在 state 里面

    在一个组件里，引入Vuex提供的Api，方便我们操作 Vuex

    import {mapActions,mapMutations,mapState,mapGetters} 'vuex'
    1
    export default {
	computed: {
		...mapState('<模块名>',[ '<state里面的数据名1>' , '<state里面的数据名2>' , ………… ]),
		...mapGetters('<模块名>',[ '<getters里面的计算属性名1' , '<getters里面的数据名2>' , ………… ])
	},
	methods: {
		...mapActions('<模块名>',[ '<actions里面的方法名1>' , '<actions里面的方法名2>' , ………… ]),
		...mapMutations('<模块名>',[ '<mutations里面的方法名1'] , '<mutations里面的方法名2'] , ………… )
	}
}

    • actions 方法有两个形参：

      第一个是 上下文对象 context，能拿到commit去提交，也能拿到 state 等

      第二个是 触发事件传过来的值 value，如果不传，默认是事件对象 event

    • mutations 方法有两个形参：

      第一个是 state 对象，里面能拿到 state 所有的数据

      第二个是 触发事件传过来的值 value ，如果不传，默认是事件对象 event

    • getters 计算属性 方法有一个形参：

      是 state 对象，里面能拿到 state 所有的 数据
